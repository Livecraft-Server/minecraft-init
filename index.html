Minecraft-init

Init script for Minecraft, Bukkit, and Spigot servers. Sponge support coming soon.

View project on GitHub

About minecraft-init

Running a Java-powered application as a service on Linux can be a pain. Often, many have done things such as running it in Byobu, leaving a desktop with a window running, or even having a Windows-based PC run their Minecraft servers. This is not optimal, as a headless Linux system would sweep the competition in this case! Unfortunately, software did not exist to solve this problem on Linux headless systems, until Ahtenus created a Minecraft init.d script. While this software was very useful, it seems to have fallen behind in supporting the latest Minecraft servers and abilities. The aim of this project is to enhance the script made by Ahtenus and provide support for newer Minecraft servers such as Sponge and the new builds of Spigot/Bukkit through the BuildTools project.

Installing

Note, this project is currently a work in progress. I do not suggest running it on a dedicated server at the moment, as the code is fairly volatile; the feature you love today may change tomorrow. I am currently working on rewriting components of the script to be more standard, as well as optimize it in order to achieve maximum stability and performance. If you are adamant about installing this on your server, the instructions are available in the README.md file.

The Future

Given that this project is currently in progress, there can and will be changes. While many often find this annoying... it's progress! The ultimate goal of this project is to accomplish the following:

  1. Allow multiple "profiles" for the user to switch between servers, or run multiple at the same time
  2. Provide an API for control panels, perhaps through a separate package
  3. Sandbox servers from one another to prevent insecure plugins from escaping the plugins folder
  4. Support various servers, such as but not limited to...
    • MCPC+
    • Cauldron
    • Sponge
    • Glowstone
    • Forge Server
    • Spigot (through BuildTools)
    • Bukkit (through BuildTools)
  5. Implement an auto-updating mechanism for the aforementioned server softwares
  6. Implement a plugin/mod updating mechanism, where links can be provided (if possible)
  7. Re-implement buggy features in the original script, such as the backup system and the ramdisk
